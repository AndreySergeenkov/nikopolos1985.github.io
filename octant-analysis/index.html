Полный разбор проектов Octant по всем эпохам

Сначала хотел собрать полный список проектов с разбивкой по эпохам, точным суммам в ETH (allocated и matched отдельно), и адресам получателей, чтобы можно было их сравнить и разбить на категории.

Octant в docs и dashboard публикует агрегированные цифры по эпохам и leaderboard на каждую эпоху отдельно, но единого dataset со всеми проектами и точными суммами по всей истории я не нашёл. Нужно было собрать самостоятельно.

Начал с того, что открыл Octant app в браузере, в DevTools Network tab переключил эпоху и поймал XHR-запросы, которые подтягивают leaderboard. Базовый URL оказался backend.mainnet.octant.app. Открыл /openapi.json и получил Swagger spec со всеми endpoint'ами V1 API. API публичный, без авторизации, работает поверх Ethereum subgraph'а.

Написал Python script с тремя нужными endpoint'ами (leaderboard на эпоху, aggregate stats, address-to-name mapping), прошёл по всем 12 эпохам, сохранил JSON.

Все эпохи кроме нулевой вернули чистые данные. Epoch 0 в API отсутствовала, потому что была pre-launch curated round до запуска V1 механики и в схему API не вписывается.

Octant считает эпохи с нуля. От Epoch 0 до Epoch 11 прошло 12 раундов. Текущий запуск, первый на новой версии платформы, называется Epoch 12. (надо проверить как он точно называется — возможно «12 раунд» или что-то такое.)

Чтобы закрыть gap по нулевой эпохе, я нашёл в документации Golem Foundation distribution wallet 0x70a0a7be87deb51e1fab16d4f2bf00be1510e476 и выгрузил через Python все его транзакции. Получил недостающие данные, заодно валидировал данные из двух источников между собой.

Дальше — разбивка проектов по категориям (research, infra, public goods, education…), визуализация распределения allocated vs matched по эпохам, и анализ повторяемости проектов между раундами.
